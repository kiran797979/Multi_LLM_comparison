## 1. Project Title

**AI System for Personalized Content Creation in Media**

---

## 2. Objective

The objective of this project is to explore and compare multiple Large Language Models (LLMs) by building a simple AI-based content generation system. The system takes a user-provided idea and converts it into structured promotional content using different LLMs, enabling a side-by-side comparison of their outputs, writing styles, and overall quality.

| LLM APIs             | Communication with hosted language models    |

---

## 4. Models Used

The following three Large Language Models were selected for comparison. All models were accessed through the OpenRouter API, which provides a single endpoint for interacting with models from different providers.

| # | Model Name             | Provider       | Access Method |
|---|------------------------|----------------|---------------|
| 1 | DeepSeek Chat          | DeepSeek       | OpenRouter    |
| 2 | Google Gemma-2-9B-IT   | Google         | OpenRouter    |
| 3 | Mistral-7B-Instruct    | Mistral AI     | OpenRouter    |

---

## 5. Prompt Used

A single, consistent prompt was used across all three models to ensure a fair and meaningful comparison:

> *"Write a promotional Instagram ad for an AI-powered study planner app for college students. Keep it energetic and include 3 hashtags."*

This prompt was carefully designed to test each model's ability to generate creative, structured, and audience-appropriate marketing content.

---

## 6. Implementation Overview

The project was implemented in the following stages:

1. **API Integration:** The OpenRouter API was configured to provide unified access to all three LLMs using a single API key and endpoint. Environment variables were used to securely store API credentials.

2. **Multi-Model Content Generation:** A Python script was developed to send the same prompt to each of the three models and collect their respective responses.

3. **Response Comparison:** The outputs from all three models were displayed together, allowing for a direct comparison of content quality, structure, tone, and formatting.

4. **Web Interface Development:** A Streamlit-based web application was built to provide a user-friendly interface. Users can select a model, enter a custom prompt, and generate content dynamically through the browser.

5. **Documentation:** All results, observations, and findings were documented for evaluation and future reference.

---

## 7. Comparative Evaluation of LLM Responses

The responses generated by each model were evaluated based on the following criteria: content relevance, tone, structure, conciseness, and adherence to the prompt instructions.

| Criteria             | DeepSeek Chat         | Google Gemma-2-9B     | Mistral-7B            |
|----------------------|-----------------------|-----------------------|-----------------------|
| **Conciseness**      | High                  | Moderate              | Moderate              |
| **Structure**        | Well-structured       | Balanced              | Slightly verbose      |
| **Tone**             | Energetic, direct     | Clear, professional   | Descriptive, coherent |
| **Hashtag Inclusion**| Yes                   | Yes                   | Yes                   |
| **Overall Quality**  | Strong                | Strong                | Good                  |

**Key Observations:**

- **DeepSeek Chat** produced the most concise and well-structured output, closely following the prompt requirements with minimal extraneous content.
- **Google Gemma-2-9B** generated balanced and clear promotional content with a professional tone suitable for marketing purposes.
- **Mistral-7B** produced slightly more descriptive output that, while coherent and well-written, was more verbose than the other two models.
- All three models demonstrated clear differences in verbosity, formatting preferences, and content style, highlighting the importance of model selection based on use case.
- Modern API-based LLMs significantly outperform older, locally-run models for structured content generation tasks.

Through this milestone, the following key learnings were gained:

- **Prompt Engineering:** Learned how prompt structure, specificity, and wording directly influence the quality and relevance of LLM-generated content.

## 10. Conclusion

Milestone 1 of the project was successfully completed. A functional AI-based content generation system was built, integrating three different Large Language Models through the OpenRouter API. The system enables users to generate and compare promotional content across models using a simple web interface.

The exploration phase demonstrated that different LLMs exhibit distinct strengths in content generation. DeepSeek excelled in conciseness, Gemma in balanced clarity, and Mistral in descriptive depth. These findings will guide model selection and prompt optimization in subsequent milestones.

The project is ready for progression to the next phase, where more advanced features such as multi-format content generation, audience-specific customization, and enhanced evaluation metrics will be explored.
